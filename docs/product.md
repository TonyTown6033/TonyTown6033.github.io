# 过敏改善可视化助手 · 产品文档

## 一句话定位

面向皮肤过敏用户的标准化拍照量化工具，帮用户看见自己是否真的在变好。

---

## 问题

过敏类产品普遍遇到三个障碍：

- 用户凭感觉判断改善，说不清楚
- 品牌和医生拿不到标准化的改善数据
- 改善证据不足，用户难以坚持使用产品

---

## 解法

复制头发拍照计数产品的链路：

```
标准化采集 → AI 识别 → 量化评分 → 前后对比 → 改善报告
```

用户每次拍一张照片，系统输出红斑面积、红度指数、视觉炎症评分，然后和基线对比，告诉用户改善了多少。

---

## MVP 范围

**做**

- 固定部位拍照 + 质量校验
- 红斑和异常区域识别
- 视觉炎症评分（含红斑面积、红度、抓痕数）
- 基线期与当前期对比
- 周报 / 月报

**不做（第一版）**

- 医学诊断和用药建议
- 鼻炎视频评估
- 眼部扫描
- 社区和内容运营

---

## 核心指标

| 指标 | 含义 |
| --- | --- |
| `lesion_area_ratio` | 异常区域占 ROI 的比例 |
| `erythema_index` | 红度强弱 |
| `scratch_mark_count` | 抓痕数量 |
| `visual_inflammation_score` | 综合视觉炎症评分 |
| `improvement_rate` | 相对基线的改善率 |

改善率计算：

```
improvement_rate = (baseline_score - current_score) / baseline_score
```

---

## 用户流程

1. 选择问题部位
2. 完成首次拍照，开启 7~14 天基线采集
3. 每天按提醒拍照，系统输出当次评分
4. 基线结束后，报告页展示改善率和趋势图

---

## 成功判断标准

MVP 跑完后，以下三条成立即值得继续：

- 用户愿意持续 14 天采集
- 重复拍摄时评分稳定（±10% 以内）
- 用户能从报告中直接感知改善

---

## 扩展路径

1. **皮肤扫描**（当前）
2. **眼部扫描**：复用拍照、质检、评分能力
3. **暴露校正**：结合花粉、尘螨数据，剔除环境干扰
4. **鼻炎视频评估**：最后扩展，链路最复杂
